DDoS insight — near-perfect scores explained
⚠
Extremely imbalanced dataset: 94,970 attack flows vs only 9 normal flows. Any model that predicts "attack" for everything achieves near-perfect recall. The F1=1.0 scores reflect this class imbalance, not necessarily model strength.
✓
Uniform packet signature: DDoS flows have zero variance in packet size (228.9–229.0 bytes) and purely unidirectional traffic. These features are trivially separable — even Logistic Regression achieves perfect separation.
→
Recommendation: Validate DDoS models on a balanced holdout set or under-sample the attack class (e.g. 50/50) to get meaningful benchmark scores. The UNSW-NB15 results (49.5% attack rate) are the more realistic performance signal.

recommended next steps
1 · Tune the best model
XGBoost leads on UNSW-NB15 with the best F1/AUC balance and trains in 2s. Use GridSearchCV or Optuna to tune:
n_estimators: 100–500
max_depth: 4–10
learning_rate: 0.01–0.3
subsample: 0.6–1.0
2 · Fix DDoS imbalance
The 99.99% attack rate inflates all DDoS scores. Fix before trusting those numbers:
• SMOTE — synthesize normal samples
• RandomUnderSampler — downsample attack class to 50/50
• class_weight='balanced' — quick fix for LR/RF
• scale_pos_weight — XGBoost equivalent
3 · Cross-dataset generalisation
The two datasets have different feature spaces. For a unified model:
• Map to a shared feature set (flow rate, packet size, duration, bytes)
• Use domain adaptation or transfer learning
• Train on UNSW, test on DDoS to measure out-of-distribution robustness
• Add a multi-label output: binary + attack category
