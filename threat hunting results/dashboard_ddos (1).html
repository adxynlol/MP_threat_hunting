Detection Rule Analysis
Layer 1 — Per-flow rules evaluate each flow individually. Priority order: DNS_Tunneling → DoS rules → Recon rules → Normal → Other. The rule that fires first wins; remaining rules are skipped.
⚡ High_Rate_Sustained_Flood
sload > 500,000 B/s dur > 0 total_pkts > 1
T1498 — Network Denial of Service
Primary rule for this dataset. Fires on 99.8% of flows. Matches extreme source load sustained over any nonzero time window. UNSW calibration: DoS sload p75 = 1.6M vs Exploits p75 = 43K.
🌊 SYN_Flood
spkts > 8 dpkts == 0 sload > 8,000 B/s
T1498.001 — Direct Network Flood
Requires server silence (dpkts=0) + many source packets. Does NOT fire here because 99.8% of flows have spkts=2. Calibrated from UNSW: 29.6% of DoS rows have dpkts=0 vs 5.8% for Exploits.
💥 Large_Burst_Short
smean > 120 B spkts > 10 dur < 1.0 s spkts > dpkts
T1498 — Network Denial of Service
Targets burst DoS with large packets. smean=229B clears the 120B threshold, but spkts=2 fails the >10 requirement. UNSW: DoS smean p50=100 vs Recon p50=56; ceiling at 120 separates cleanly.
📡 UDP_ICMP_Flood
proto = 1 or 17 spkts > 10 sload > 100,000 B/s dur > 0.0001
T1498.002 — Reflection Amplification
Targets UDP (proto=17) and ICMP (proto=1) amplification/reflection floods. Dataset lacks explicit proto column — in PCAP mode, proto is extracted from packet headers. Requires multiple packets to exclude noise.
Layer 2 — Cross-flow correlation rules run as a second pass over the full table, upgrading flows where single-flow rules missed the broader campaign pattern. Applied via apply_cross_flow_rules(df) in pcap mode only.
🎯 DDoS_Volume_Campaign
pair total sbytes ≥ 100,000,000 per src→dst pair
T1498 — Network Denial of Service
Groups all flows by (src_ip, dst_ip) and sums sbytes. If any pair exceeds 100 MB, every flow in that pair is upgraded to DoS. Threshold set high (100 MB) to avoid flagging legitimate CDN/streaming traffic.
💣 SYN_Flood_Campaign
≥15 half-open flows (dpkts=0) same src→dst pair dport < 10,000
T1498.001 — Direct Network Flood
Counts SYN-only (no ACK) flows per source-target pair. 15+ half-open connections = sustained SYN flood. The dport < 10,000 guard prevents WebRTC/push connections from server to client ephemeral ports from triggering.
🔍 Port_Scan_Campaign
≥10 distinct dst ports from same src_ip spkts ≤ 15, sbytes ≤ 1,200 dport < 10,000
T1595 — Active Scanning
Counts unique server ports probed by each source. Scans qualifying flows only (small, short). The dport < 10,000 ceiling prevents HTTP/HTTPS browsing — which naturally hits many ports — from looking like port scanning.
🌐 IP_Sweep
≥8 distinct dst IPs from same src_ip spkts ≤ 5, sbytes ≤ 500 dur < 1.0 s
T1595.001 — Scanning IP Blocks
Detects horizontal scanning across hosts. Small, fast probes only. Only upgrades rows not already flagged as Recon by Layer 1. Complementary to Port_Scan_Campaign which is vertical (many ports, one host).
