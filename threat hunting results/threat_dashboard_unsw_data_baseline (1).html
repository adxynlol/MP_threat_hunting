Empirical Data Evidence (The "Why" Behind the Rules)
Mean Pkt Size (smean) Distribution
Data mapping that proved smean ≤ 90 successfully captures most Recon probes without triggering on heavier DoS payloads.
Responder Pkts (dpkts) in DoS
Nearly 30% of UNSW DoS traffic receives exactly 0 responder packets. This data drove the dpkts == 0 SYN_Flood rule threshold.
Originator Pkts (spkts) in Recon
85% of UNSW Recon flows use exactly 10 source packets (standard Nmap default). This data drove the spkts 8–15 rule.
Layer 1 Per-Flow Calibrated Rules
🔍 Port_Scan_Probe
spkts 8–15 sbytes ≤ 800 dport NOT IN (80,443,22)
T1595.002 — Vulnerability Scanning
Data Basis: Built directly from the empirical chart above showing 85% of Recon rows sitting precisely at spkts=10. Filtering well-known service ports prevents misclassifying normal web handshakes.
👻 Stealth_Scan_SubSecond
smean 40–90 dur < 1.5s spkts ≥ 8
T1595.001 — Scanning IP Blocks
Data Basis: Built from the smean distribution curve. The smean 40-90 bounds capture symmetric, small-packet SYN or Ping sweeps, definitively isolating them from DoS traffic.
🌊 SYN_Flood
dpkts == 0 spkts > 8 sload > 8000
T1498.001 — Direct Network Flood
Data Basis: The Responder Packets chart above proved nearly 30% of DoS traffic in this dataset had dpkts == 0, confirming overwhelming asymmetric traffic.
🕵️ DNS_Tunneling
service == 'dns' sbytes > 5000 OR smean > 200
T1071.004 — App Layer Protocol (DNS)
Data Basis: Legitimate DNS queries are tiny (<512B). Tunneling tools (like iodine) encode data in subdomains, generating massive payload overhead isolated by smean > 200.
